# Actividad 1.1: Investigación de Conceptos Fundamentales

## ¿Qué es la ciencia de datos?

La ciencia de datos es un campo interdisciplinario que utiliza métodos científicos, procesos, algoritmos y sistemas para extraer conocimiento e insights de datos estructurados y no estructurados. Sus componentes principales son:

- Estadística y matemáticas
- Programación y habilidades técnicas (Hacking Skills)
- Conocimiento del dominio (Substantive Expertise)

Según el diagrama de Venn de Drew Conway (2010), la ciencia de datos se encuentra en la intersección de estas tres áreas.

## Diferencia entre datos estructurados y no estructurados

| Característica | Datos Estructurados | Datos No Estructurados |
|---------------|---------------------|----------------------|
| Formato | Tablas, filas y columnas | Sin formato predefinido |
| Ejemplos | Bases de datos SQL, hojas de cálculo, CSV | Imágenes, videos, correos, reseñas de texto |
| Almacenamiento | Bases de datos relacionales | Data lakes, bases NoSQL |
| Consulta | SQL, consultas directas | Requiere procesamiento previo (NLP, visión computacional) |

## Las 5 V del Big Data

1. **Volumen**: cantidad masiva de datos generados (ej: millones de transacciones diarias en un e-commerce).
2. **Velocidad**: rapidez con la que se generan y procesan los datos (ej: streaming de eventos en tiempo real).
3. **Variedad**: diferentes tipos y formatos de datos (ej: CSV, JSON, imágenes, logs).
4. **Veracidad**: calidad y confiabilidad de los datos (ej: datos duplicados o incompletos que requieren limpieza).
5. **Valor**: capacidad de transformar datos en insights accionables (ej: modelos predictivos que optimizan inventario).

## Perfiles profesionales en ciencia de datos

- Data Scientist: modelos predictivos, análisis exploratorio, machine learning.
- Data Engineer: pipelines de datos, infraestructura, ETL/ELT.
- Data Analyst: reportes, dashboards, consultas SQL, KPIs.
- ML Engineer: despliegue de modelos a producción, MLOps.
- Business Analyst: puente entre negocio y equipo técnico, requerimientos.
- ETL Developer: procesos de extracción, transformación y carga de datos.
- Big Data Architect: diseño de arquitectura global de datos, gobernanza.
