Los ejercicios complementarios te proporcionan la base de bases de datos y programación necesaria para:

1. **Comprender SQL** y las consultas a bases de datos
2. **Manejar estructuras JSON** que son fundamentales en MongoDB

4. **Diseñar modelos de datos** NoSQL efectivos

> **Justificación:** Estos conocimientos son prerrequisitos básicos que facilitan la comprensión de arquitecturas de almacenamiento y MongoDB. Sin esta base, será más difícil completar las actividades prácticas de manera efectiva.

## Actividades Propuestas

### Actividad 2.1: Investigación de Arquitecturas de Datos

**Descripción:** Investiga las diferentes arquitecturas de almacenamiento de datos.

**Instrucciones:**
1. Investiga qué son los Data Warehouses y Data Lakes
2. Compara las características de cada uno
3. Investiga qué es un Data Mart
4. Crea un diagrama mostrando las diferencias

**Carpeta de entrega:** `Semana2/Actividades/Actividad2.1/`

**Git:**
```bash
git add Semana2/Actividades/Actividad2.1/
git commit -m "Semana2: Actividad 2.1 - Arquitecturas de datos"
git push
```

---

### Actividad 2.2: Introducción a MongoDB

**Descripción:** Instala y configura MongoDB en tu entorno local.

**Instrucciones:**
1. Instala MongoDB Community Server
2. Instala MongoDB Compass (interfaz gráfica)
3. Crea una base de datos de prueba
4. Crea una colección y agrega 5 documentos de ejemplo

**Carpeta de entrega:** `Semana2/Actividades/Actividad2.2/`

**Git:**
```bash
git add Semana2/Actividades/Actividad2.2/
git commit -m "Semana2: Actividad 2.2 - Introducción a MongoDB"
git push
```

---

### Actividad 2.3: Operaciones CRUD en MongoDB

**Descripción:** Practica las operaciones básicas de MongoDB.

**Instrucciones:**
1. Conecta Python a MongoDB usando pymongo
2. Crea una colección llamada "empleados" con campos: nombre, departamento, salario
3. Inserta al menos 10 documentos
4. Realiza las siguientes operaciones:
   - READ: Consulta empleados de un departamento específico
   - UPDATE: Actualiza el salario de un empleado
   - DELETE: Elimina un empleado
5. Muestra los resultados de cada operación

**Carpeta de entrega:** `Semana2/Actividades/Actividad2.3/`

**Entregable:** Script de Python con todas las operaciones

**Git:**
```bash
git add Semana2/Actividades/Actividad2.3/
git commit -m "Semana2: Actividad 2.3 - Operaciones CRUD"
git push
```

---

### Actividad 2.4: Modelado de Datos NoSQL

**Descripción:** Diseña un modelo de datos para un caso específico.

**Instrucciones:**
1. Elige un caso: "Sistema de biblioteca digital"
2. Diseña la estructura de documentos necesaria
3. Crea al menos 5 colecciones con relaciones
4. Define los campos y tipos de datos
5. Justifica tu diseño

**Carpeta de entrega:** `Semana2/Actividades/Actividad2.4/`

**Git:**
```bash
git add Semana2/Actividades/Actividad2.4/
git commit -m "Semana2: Actividad 2.4 - Modelado NoSQL"
git push
```
